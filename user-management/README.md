## User Management API

### Описание
API для управления пользователями и аутентификации в системе.

### Эндпоинты

#### 1. Создать нового пользователя
**POST** `/users/create`
- **Описание:** Создаёт нового пользователя
- **Коды ответа:**
    - `201 Created` - Пользователь успешно создан
    - `400 Bad Request` - Не удалось создать пользователя

#### 2. Получить список всех пользователей
**GET** `/users`
- **Описание:** Возвращает список всех пользователей
- **Коды ответа:**
    - `200 OK` - Успешно получен список пользователей

#### 3. Получить пользователя по email
**GET** `/users/mail/{email}`
- **Описание:** Возвращает пользователя по email
- **Заголовки:**
    - `Authorization: Bearer {token}` (обязательный)
- **Коды ответа:**
    - `200 OK` - Пользователь найден
    - `401 Unauthorized` - Пользователь не авторизован
    - `403 Forbidden` - Доступ запрещён

#### 4. Получить пользователя по id
**GET** `/users/{id}`
- **Описание:** Возвращает пользователя по ID
- **Коды ответа:**
    - `202 Accepted` - Пользователь найден
    - `500 Internal Server Error` - Пользователь не найден

#### 5. Обновить данные пользователя по id
**PUT** `/users/update/{id}`
- **Описание:** Обновляет данные пользователя по ID
- **Коды ответа:**
    - `202 Accepted` - Пользователь успешно обновлён
    - `400 Bad Request` - Не удалось обновить данные пользователя

#### 6. Удалить пользователя по id
**DELETE** `/users/delete/{id}`
- **Описание:** Удаляет пользователя по ID
- **Коды ответа:**
    - `202 Accepted` - Пользователь успешно удалён
    - `400 Bad Request` - Не удалось удалить пользователя

#### 7. Получить JWT по email и password
**POST** `/login`
- **Описание:** Возвращает JWT по email и password
- **Заголовки:**
  - `Authorization: Basic {encodedAuthData}` (обязательный)
- **Коды ответа:**
  - `200 OK` - Пользователь найден
  - `401 Unauthorized` - Неверные логин или пароль
  - `500 Internal Server Error` - Отсутствует заголовок авторизации
